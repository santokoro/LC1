Лексический анализатор (сканер)
───────────────────────────────────────────────────────────────
Сканер реализован вручную в MainWindow.cs.
Он выделяет токены следующих типов:

Keyword — ключевые слова

Identifier — идентификаторы

UnsignedInteger — целые числа

RealNumber — вещественные числа

Assignment (=)

Colon (:)

StatementEnd (;)

Minus (для унарного минуса)

Error — недопустимые символы

Ошибочные токены автоматически добавляются в таблицу токенов.

Синтаксическая грамматика (КС‑грамматика)
───────────────────────────────────────────────────────────────
Грамматика G⟨START⟩:

Код
Программа → { Объявление }

Объявление → 'const' 'val' Идентификатор Тип '=' Выражение ';'

Тип → ':' 'Double'

Выражение → '-' Число | Число

Число → UnsignedInteger | RealNumber
Грамматика является:

контекстно‑свободной

детерминированной

без левой рекурсии

пригодной для рекурсивного спуска

Алгоритм нейтрализации ошибок (метод Айронса)
───────────────────────────────────────────────────────────────
Реализован в виде:

флага panicMode

функции SkipTo(...)

восстановления до ближайшего ;

продолжения разбора после ошибки

При ошибке:

фиксируется сообщение

парсер переходит в режим паники

пропускает токены до синхронизирующего символа

продолжает разбор

Это позволяет:

не прекращать анализ после первой ошибки

собрать таблицу всех ошибок

построить AST даже при ошибках
