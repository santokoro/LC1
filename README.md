# LC1 — языковой процессор (лексика, синтаксис, AST, семантика)

**Название работы:** Построение AST и проверка контекстно-зависимых условий

**Автор:** Гидульянов Кирилл Сергеевич  
**Группа:** АВТ‑313  

**Вариант №54.** Объявление вещественной константы с инициализацией на языке Kotlin (подмножество).

**Цель работы**
Изучить назначение и принципы работы семантического анализатора в структуре компилятора. Освоить методы построения абстрактного синтаксического дерева (AST) и проверки контекстно-зависимых условий (семантических правил) для заданной синтаксической конструкции.

**Постановка задачи**
Развить ранее созданный синтаксический анализатор (парсер) до семантического: построить абстрактное синтаксическое дерево (AST) и реализовать проверку контекстно-зависимых условий в соответствии с индивидуальным вариантом курсовой работы.

**Контекстно-зависимые условия:** Только Правило 1 (уникальность идентификаторов): Проверить, что имя не было объявлено ранее в той же области видимости.
Ввод:
const val a : Double = 1.0;
const val a : Double = 2.0;

Ожидаемый результат:
Повторное объявление идентификатора в той же области видимости

**Структура AST**
1. Описание типов узлов:

Базовый класс AstNode
Общий предок всех узлов AST. Хранит позицию во входном тексте: номер строки, начальный и конечный столбец. Задаёт имя типа узла (NodeTypeName) и список дочерних узлов (GetChildren()).

ProgramNode
Корень дерева всей программы. Содержит упорядоченный список объявлений констант. Дочерние узлы — только узлы ConstDeclNode (по одному на каждое объявление в исходном тексте).

ConstDeclNode
Одно объявление вида const val имя : Double = инициализатор ;. В полях хранятся: отдельные узлы для ключевых слов const и val, узел имени объявляемой константы, узел типа Double, а также узел инициализатора (поддерево справа от =). Дочерние узлы в фиксированном порядке: сначала const, затем val, затем идентификатор объявления, затем тип, затем инициализатор.

KeywordModifierNode
Одно ключевое слово-модификатор (const или val) в виде строки в поле. Дочерних узлов нет.

IdentifierNode
Имя объявляемой константы (лексема-идентификатор слева от :). Дочерних узлов нет.

DoubleTypeNode
Объявленный тип (в варианте задания — Double, имя типа хранится в поле). Дочерних узлов нет.

DoubleLiteralNode
Вещественный литерал в инициализаторе: числовое значение после разбора и исходный текст из программы. Дочерних узлов нет.


**Графические средства**
Использованные графические средства

Для наглядного представления абстрактного синтаксического дерева в программе используется платформа WPF (Windows Presentation Foundation): отдельное окно визуализации AST построено как пользовательский интерфейс на XAML, а отрисовка выполняется на элементе Canvas. Узлы дерева отображаются в виде прямоугольных блоков (Border) с текстовыми подписями (название типа узла и краткое содержимое). Связи «родитель — потомок» изображаются линиями (Line) и наконечниками стрелок (Polygon). Расположение узлов по уровням и маршрутизация рёбер (в том числе ортогональные участки, чтобы линии не пересекали прямоугольники соседних узлов) реализованы в коде класса AstGraphRenderer. Окно открывается из главного приложения после успешного синтаксического анализа исходного текста, когда уже построен объект программы в памяти (ProgramNode).
